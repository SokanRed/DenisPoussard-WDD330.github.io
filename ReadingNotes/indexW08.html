▼ Chapter 8 | CSS3 Transforms and Transitions

Transforms | mdn CSS Transforms

The CSS3 transform lets translate, rotate, scale and/or skew any element on the page.

Translate
transform: translateX(45px);
transform: translateY(-30px);
Rotate
transform: rotate(10deg)
Scale
transform: scale(1.5, 0.25);
Skew
transform: skew(15deg, 4deg);

Transitions | mdn CSS Transitions

Steps to create a simple transition using only CSS :

  1. Declare the original state of the element
  2. Declare the final state of the element (example, a :hover state)
  3. Include the transition functions in the default style declaration

General Rule is : "If there is a valid midpoint, it can be transitioned"

Exception to this rule is visibility

The list of properties that can be animated is found at : Animated Properties

Remember of Supporting Older Browsers

The transition-duration sets how long the transition will take.

The transition-timing-function lets control the pace of the transition.

Timing function can be describe more precisely by defining cubic-bezier function. Useful link : cubic-bezier

Another document to set the timing function and time to watch it: Animation Cubic Bezier

The transition-delay Property introduces a delay before the transition begins.

Animations mdn CSS Animations

CSS animations, unlike transitions, allows to control each step of an animation via keyframes.

For an animation called myAnimation, the @keyframes rule would look like this (do not quote the animation name):

@-webkit-keyframes myAnimation {
/* put animation keyframes here */
}
@keyframes myAnimation {
/* put animation keyframes here */
}

Few simple animations :

@keyframes moveRight {
from {
transform: translateX(-50%);
}
to {
transform: translateX(50%);
}
}
@keyframes appearDisappear {
0%, 100% {
opacity: 0;
}
20%, 80% {
opacity: 1;
}
}
@keyframes bgMove {
100% {
background-position: 120% 0;
}
}

Moving On

Remember : "Just Because You Can, Doesn't Mean You Should"

Use animations and transitions where it makes sense, enhancing the user experience—and skip it everywhere else.

▼ Chapter 12 | Canvas, SVG, and Drag and Drop

Canvas | mdn Canvas API

The Canvas API provides a means for drawing graphics via JavaScript and the HTML canvas element. Among other things, it can be used for animation, game graphics, data visualization, photo manipulation, and real-time video processing.

Further Reading : "HTML5 Canvas -- the Basics" at Dev.Opera & Safari's HTML Canvas Guide

Comments :

Every time you finish drawing a shape, the canvas no longer has access to that shape. Still, canvas does allow you to save the images you create to a PNG or JPEG file. There’s also no access to what’s drawn on the canvas via the DOM. Because of this, canvas is much faster than SVG.

SVG | mdn SVG

SVG stands for Scalable Vector Graphics, a specific file format to describe vector graphics using XML.

XML stands for eXtensible Markup Language.

Drawing a circle in SVG is arguably easier than drawing a circle with canvas.

Image editors to help Inkscape Draw Freely & OpenClipArt

Comments :

SVG is accessible via the DOM. You can use Adobe Illustrator and obtain the SVG code instantly

In summary, if you need to paint pixels to the screen and have no concerns about the ability to retrieve and modify your shapes, canvas is probably the better choice. If, on the other hand, you need to be able to access and change specific aspects of your graphics, SVG might be more appropriate.

Drag and Drop | mdn HTML Drag and Drop API

The Drag and Drop API allows to specify that certain elements are draggable, and then specify what should happen when these draggable elements are dragged over or dropped onto others elements on the page.
